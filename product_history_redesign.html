How this page feeds your dashboard classification
Step 1 — compute velocity for every product in one pass
Build one shared function that both this page and the dashboard call. No duplication. Currently your renderStockIntel() and renderVendorProductAnalytics() both recompute velocity independently — that's the same logic written twice.
// One shared function — call from both dashboard and this page function buildProductAnalytics(periodDays = 30) { const now = Date.now(); const periodMs = periodDays * 86400000; return products.map(p => { const recentSales = sale_items.filter(i => (i.product_id === p.id || i.product_name === p.name) && (now - new Date(i.created_at).getTime()) <= periodMs ); const soldUnits = recentSales.reduce((s, i) => s + i.qty, 0); const velocity = soldUnits / periodDays; // units per day const stockLeft = batches .filter(b => b.product_id === p.id) .reduce((s, b) => s + b.quantity, 0); const daysOfSupply = velocity > 0 ? Math.round(stockLeft / velocity) : 999; const maxBatchAge = Math.max(...batches .filter(b => b.product_id === p.id && b.created_at) .map(b => Math.floor((now - new Date(b.created_at).getTime()) / 86400000)), 0); return { product: p, velocity, soldUnits, stockLeft, daysOfSupply, maxBatchAge }; }); }
Step 2 — classify using relative thresholds, not fixed numbers
After building analytics, compute catalog average velocity, then classify each product relative to that average. This auto-adjusts to your shop size — no hardcoded cutoffs.
function classifyProducts(analytics) { const velocities = analytics.map(a => a.velocity).filter(v => v > 0); const avgVelocity = velocities.reduce((s, v) => s + v, 0) / (velocities.length || 1); return analytics.map(a => { let tag; if (a.velocity >= avgVelocity * 1.5) tag = 'high'; // 1.5× avg = fast else if (a.velocity >= avgVelocity * 0.5) tag = 'normal'; // 0.5–1.5× avg else if (a.velocity > 0) tag = 'low'; // below half avg else if (a.maxBatchAge >= 30) tag = 'dead'; // no sales + old else tag = 'new'; // no sales yet, recent // Old stock: aged batch + majority unsold (need original_qty stored) const isOldStock = a.maxBatchAge >= 30 && a.stockLeft > 0 && a.velocity < avgVelocity * 0.3; return { ...a, tag, isOldStock }; }); }
Step 3 — dashboard reads classifications, doesn't recompute
Dashboard just filters the already-classified array. Simple, fast, single source of truth.
// On dashboard render: const analytics = buildProductAnalytics(30); const classified = classifyProducts(analytics); const highSelling = classified.filter(a => a.tag === 'high'); const lowSelling = classified.filter(a => a.tag === 'low' || a.tag === 'dead'); const oldStock = classified.filter(a => a.isOldStock); // Product history page — same data, just one product: function openProductHistory(productId) { const analytics = buildProductAnalytics(30); const classified = classifyProducts(analytics); const thisProduct = classified.find(a => a.product.id === productId); // render the page using thisProduct data }
One thing to add to your batch data model: store original_quantity when a batch is created. Right now you only have quantity (current remaining). Without original_quantity you can't compute "what % of this batch is unsold" — which is needed for old stock detection. One field, added at purchase time, unlocks the full classification.
